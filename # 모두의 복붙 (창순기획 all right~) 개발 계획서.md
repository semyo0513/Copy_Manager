# 모두의 복붙 (창순기획 all right~) 개발 계획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **프로그램명** | 모두의 복붙 (창순기획) |
| **목표** | 엑셀 데이터를 웹페이지 폼에 자동 붙여넣고 탭(Tab) 이동을 수행하는 Windows 실행 파일(.exe) 제작 |
| **핵심 가치** | 단순 반복 입력 업무의 완전 자동화, 직관적이고 다채로운 UI/UX, 안전한 매크로 제어 |

## 2. 기술 스택 (추천)

웹앱 개발에 익숙한 특성을 살려, 화려한 UI와 애니메이션 구현에 유리한 웹 프론트엔드 기술을 데스크톱 앱에 결합하는 방식을 채택합니다.

| 구분 | 기술 / 라이브러리 | 용도 및 채택 사유 |
|---|---|---|
| **언어** | Python 3 | 매크로 제어 및 데스크톱 애플리케이션 개발에 최적화 |
| **UI/UX** | Eel (HTML/CSS/JS) | 다채로운 디자인 및 CSS/JS 기반의 화려한 반응형 애니메이션 구현 용이 |
| **자동화 제어** | PyAutoGUI | 붙여넣기(`Ctrl+V`), `Tab` 키 입력 등 시스템 키보드 이벤트 시뮬레이션 |
| **단축키/안전** | Keyboard | 글로벌 단축키 인식을 통한 매크로 긴급 중지(Kill Switch) 기능 구현 |
| **클립보드** | Pyperclip | 엑셀에서 복사된 다중 행/열 데이터(구분자 `\t`, `\n`) 읽기 및 파싱 |
| **빌드/배포** | PyInstaller | Python 스크립트와 웹 에셋을 단일 실행 파일(.exe)로 패키징 |

## 3. 핵심 기능 명세

### 3.1. 데이터 파싱 및 입력 로직
*   **데이터 인식**: 사용자가 엑셀에서 영역을 복사한 상태에서 프로그램 실행 시, 클립보드 데이터를 즉시 읽어와 행과 열 단위의 배열로 자동 파싱.
*   **실행 시퀀스**:
    1. 배열의 첫 번째 셀 데이터를 클립보드에 장전 후 `Ctrl + V` 입력.
    2. 사용자가 지정한 횟수만큼 `Tab` 키 입력 (다음 입력 필드로 포커스 이동).
    3. 지연 시간(Delay) 대기 후 다음 셀 데이터로 이동하여 루프 반복.

### 3.2. 매크로 안전장치 및 설정
*   **긴급 중지(Kill Switch)**: 매크로가 실행되는 동안 포커스가 엉뚱한 곳으로 튀는 것을 방지하기 위해, 언제든 `ESC` 또는 `F12` 키를 누르면 즉시 작동이 중단되도록 설계.
*   **지연 시간(Delay) 제어**: 웹페이지의 반응 속도를 고려하여 입력과 탭 이동 사이의 딜레이(예: 0.1초 ~ 1.0초)를 조절하는 옵션 제공.

## 4. UI/UX 및 애니메이션 기획

*   **메인 레이아웃**:
    *   **상단 헤더**: "창순기획 all right~" 타이포그래피에 부드러운 네온 글로우(Neon Glow) 애니메이션 적용.
    *   **중앙 컨트롤러**: 탭(Tab) 횟수 조절용 스피너(+, - 버튼 클릭 시 통통 튀는 바운스 효과), 복사된 데이터가 몇 줄인지 보여주는 상태 표시창.
    *   **하단 액션 버튼**: 대형 그라데이션 "실행하기" 버튼. 마우스 호버 시 색상이 흐르듯 변하며 클릭 시 물결(Ripple) 이펙트 발생.
*   **작업 진행 상태 시각화**:
    *   **대기 모드**: 버튼에 펄스(Pulse) 애니메이션을 주어 사용자 액션 유도.
    *   **실행 모드**: 프로그램 창 상단에 얇은 프로그레스 바(Progress Bar)가 차오르며, 남은 데이터 갯수가 카운트다운 애니메이션으로 표시됨.
    *   **피드백**: 작업 완료 또는 오류 발생 시 화면 하단에서 스르륵 올라오는 Toast 알림 제공.

## 5. 단계별 개발 일정

| 단계 | 주요 작업 내용 |
|---|---|
| **Phase 1: 코어 매크로 로직 구현** | 클립보드 파싱(Pyperclip), 자동 붙여넣기 및 Tab 루프(PyAutoGUI), 긴급 중지 단축키 매핑 테스트 |
| **Phase 2: UI/UX 개발 (웹 기반)** | HTML/CSS로 다채로운 레이아웃 구성, CSS Keyframes를 활용한 펄스/바운스 애니메이션 작성 |
| **Phase 3: 로직-UI 연동 (Eel)** | Python 백엔드와 JS 프론트엔드 통신 연결, UI에서 입력받은 Tab 횟수를 매크로 로직으로 전달 |
| **Phase 4: 최적화 및 안정성 확보** | 웹 환경 변수에 따른 입력 딜레이 튜닝, 데이터 엣지 케이스(공백, 특수문자 등) 예외 처리 |
| **Phase 5: 실행 파일 빌드** | PyInstaller 및 Auto-py-to-exe를 활용하여 의존성이 포함된 단일 `.exe` 파일 빌드 및 구동 테스트 |
